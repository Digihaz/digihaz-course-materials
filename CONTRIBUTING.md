# Contributing to digihaz-course-materials

## Branch workflow

1. Create a feature branch from `dev`: `module-XX-description`
2. Make your changes and commit
3. Open a Pull Request targeting `dev` (not `main`)
4. Request review from the Course Director

## Checklist before opening a PR

- [ ] Branch is up to date with `dev`
- [ ] Exercise notebooks have **outputs cleared** (`Cell → All Output → Clear`)
- [ ] Topic README updated (objectives, Colab badge, references)
- [ ] No files over 50 MB committed directly
- [ ] No credentials, API keys or personal data

## Never push directly to `main` or `dev`
